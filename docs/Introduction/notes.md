## What is a design pattern?
A solution to a high level problem and context, that has been proven
to be effective. It also includes understanding consequences of using a
given pattern.

## Components of a design pattern
* name
* problem
* solution
* consequences

## Pattern classifications
#### Purpose
* Creational
 * Object creation related
* Structural
 * Composition of classes / objects
* Behavioral
 * How classes / objects interact or distribute responsibility

#### Scope
* Class Patterns
 * Are static / compile time
* Object Patterns
 * Dynamic, changing

#### Principles of OO design
* Favor object composition over inheritance

#### Hot To Select A Design Pattern
* Consider how a design pattern solves a problem
* See if a patterns intent matches your own
* Know relationships between different design patterns
* Study patterns of like purpose
